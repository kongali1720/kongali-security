# Support

Thank you for using KONGALI SECURITY.

This document explains how to get help, report problems, ask questions, and find useful resources.

> **Secure. Analyze. Automate.**

---

## Before Asking for Support

Before requesting support, please check:

- `README.md`
- `docs/`
- `FAQ.md`
- `GLOSSARY.md`
- Existing GitHub Issues
- Existing GitHub Discussions
- `CONTRIBUTING.md`

Please search existing issues before opening a new one.

---

## Bug Reports

If you believe you have found a bug, please open a GitHub Issue.

Include:

- KONGALI SECURITY version
- Operating system
- Python version
- Installation method
- Relevant configuration
- Steps to reproduce
- Expected behavior
- Actual behavior
- Error messages
- Relevant logs

Please remove:

- Passwords
- API keys
- Access tokens
- Private keys
- Credentials
- Personally identifiable information
- Sensitive infrastructure details

before submitting your report.

---

## Feature Requests

Feature requests are welcome.

Please explain:

1. What problem the feature solves.
2. Why the feature would be useful.
3. How you expect it to work.
4. Potential alternatives.
5. Potential security implications.

Large architectural changes should be discussed before implementation.

---

## Security Vulnerabilities

Do not report security vulnerabilities through public GitHub Issues.

Please follow the instructions in:

[SECURITY.md](SECURITY.md)

Responsible disclosure helps protect users and the wider community.

---

## Documentation Issues

If you find incorrect, outdated, or unclear documentation, please report it through a GitHub Issue or submit a Pull Request.

Documentation improvements are highly encouraged.

---

## Pull Requests

If you have a fix or improvement, you are welcome to submit a Pull Request.

Please review:

[CONTRIBUTING.md](CONTRIBUTING.md)

before submitting your contribution.

---

## Community Discussions

For general questions, ideas, architecture discussions, and community conversations, use the appropriate GitHub Discussions or project communication channels when available.

---

## Support Scope

Community support may cover:

- Installation
- Configuration
- Basic usage
- CLI usage
- Documentation
- Bug reports
- Development questions
- Contribution guidance

Support does not include:

- Unauthorized security activities
- Attacks against third-party systems
- Illegal activity
- Requests to bypass security controls
- Assistance with malicious software
- Accessing systems without authorization

---

## Production Deployments

KONGALI SECURITY is an evolving open-source project.

Before using it in production environments, users should:

- Review the documentation.
- Review the security policy.
- Evaluate dependencies.
- Test the deployment.
- Apply appropriate security controls.
- Monitor the system.
- Maintain backups.
- Keep dependencies updated.
