# Cybersecurity Glossary

This glossary provides basic definitions of cybersecurity terms used throughout KONGALI SECURITY documentation.

---

## AI

Artificial Intelligence.

Technology used to perform tasks that traditionally require human intelligence.

---

## IOC

Indicator of Compromise.

An artifact that may indicate malicious activity or a security incident.

Examples include:

- IP addresses
- Domains
- URLs
- File hashes
- Email addresses

---

## OSINT

Open-Source Intelligence.

Information collected and analyzed from publicly available sources.

---

## Threat Intelligence

Information about threats, threat actors, indicators, tactics, techniques, and procedures used to improve security decisions.

---

## TTP

Tactics, Techniques, and Procedures.

A structured description of how threat actors operate.

---

## YARA

A rule-based pattern matching technology commonly used for identifying and classifying malware and suspicious files.

---

## STIX

Structured Threat Information Expression.

A standardized language and serialization format for representing cyber threat intelligence.

---

## TAXII

Trusted Automated Exchange of Intelligence Information.

A protocol designed to exchange cyber threat intelligence.

---

## SOC

Security Operations Center.

A team or function responsible for monitoring and responding to cybersecurity events.

---

## SIEM

Security Information and Event Management.

A system that collects, analyzes, correlates, and manages security events and logs.

---

## FIM

File Integrity Monitoring.

The process of monitoring files and detecting unauthorized changes.

---

## Incident Response

The process of identifying, investigating, containing, and recovering from security incidents.

---

## Threat Actor

An individual, group, or organization responsible for or associated with malicious cyber activity.

---

## Threat Hunting

A proactive process of searching for potential threats or suspicious activity within systems and networks.

---

## Security Automation

The use of software and workflows to automate repetitive security tasks.

---

## AI-Assisted Security

The use of artificial intelligence to assist security professionals with analysis, investigation, detection, and reporting.

---

## IOC Enrichment

The process of adding additional context and intelligence to an indicator.

---

## Hash

A fixed-length value generated from data using a hashing algorithm.

Common examples include:

- MD5
- SHA-1
- SHA-256

---

## Malware

Malicious software designed to compromise, disrupt, damage, or gain unauthorized access to systems.

---

## Vulnerability

A weakness that may be exploited to compromise a system or application.

---

## CVE

Common Vulnerabilities and Exposures.

A public identifier for known cybersecurity vulnerabilities.

---

## CVSS

Common Vulnerability Scoring System.

A framework used to assess the severity of cybersecurity vulnerabilities.

---

## RBAC

Role-Based Access Control.

An authorization model where permissions are assigned based on user roles.

---

## API

Application Programming Interface.

A defined interface that allows software components to communicate.

---

## CLI

Command-Line Interface.

A text-based interface used to interact with software.

---

## CI/CD

Continuous Integration and Continuous Delivery or Deployment.

Practices used to automate software testing, integration, and delivery.

---

## Supply Chain Security

The process of protecting software, dependencies, infrastructure, and development processes from compromise.

---

## Responsible Disclosure

A process where security vulnerabilities are privately reported to maintainers before public disclosure.

---

## Authorized Security Testing

Security testing performed with explicit permission from the system owner or authorized organization.

---

## Zero-Day

A previously unknown vulnerability or vulnerability for which no official fix may yet be available.
