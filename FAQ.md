# Frequently Asked Questions

## What is KONGALI SECURITY?

KONGALI SECURITY is an open-source cybersecurity and security automation framework.

The project is designed to bring together security analysis, threat intelligence, OSINT, monitoring, automation, reporting, and AI-assisted security workflows.

---

## Who is KONGALI SECURITY for?

The project is intended for:

- Security researchers
- SOC analysts
- Blue teams
- Security engineers
- System administrators
- Threat intelligence analysts
- Incident responders
- Cybersecurity students
- Educators
- Authorized security testing teams

---

## Is KONGALI SECURITY production ready?

The project is currently under active development.

Early versions may contain incomplete features, changing APIs, or architectural changes.

Users should carefully evaluate the project before deploying it in production environments.

The project targets modern Python versions.

## How do I report a bug?

Please open a GitHub Issue and provide enough information to reproduce the problem.

Do not include secrets or sensitive information.

---

## How do I report a security vulnerability?

Do not report security vulnerabilities through public GitHub Issues.

Follow the procedures in:

[SECURITY.md](SECURITY.md)

---

## Does KONGALI SECURITY perform unauthorized hacking?

No.

KONGALI SECURITY is intended for authorized security testing, defensive security operations, research, education, and legitimate security analysis.

Users are responsible for ensuring they have authorization to test or monitor systems.

---

## Does the project include AI?

The project roadmap includes AI-assisted security capabilities.

AI features are intended to assist security professionals and should not replace human judgment.

See:

[ROADMAP.md](ROADMAP.md)

---

## Is AI fully autonomous?

No.

The long-term design emphasizes human-in-the-loop workflows, validation, security controls, and auditability.
